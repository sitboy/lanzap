# Zap

**Один Wi‑Fi — открыл и отправил.**

AirDrop, который не нужно устанавливать, — помощник для передачи файлов без входа в аккаунт.
Устройства в одной сети открывают одну ссылку и попадают в один чат; текст и файлы передаются
**напрямую, peer‑to‑peer, никогда не касаясь сервера**.

## Возможности

- 🚀 **Без приложения, без входа, без добавления контактов** — просто откройте ссылку в браузере
- 🔒 **Файлы никогда не касаются сервера** — прямое P2P‑соединение, без ограничений по размеру, без повторного сжатия; сервер не хранит ни данных, ни логов
- 📁 **Целые папки, структура сохраняется** — перетащите или выберите; в десктопном Chromium дерево папок пишется прямо на диск, остальные браузеры получают ZIP
- 👥 **Групповые и приватные чаты** — одна комната «Everyone» плюс приватный диалог с каждым устройством
- 📷 **Три способа подключиться** — автоматическое обнаружение в одной сети / сканирование QR‑кода прямо на странице / 5‑символьный код комнаты / ссылка для отправки
- 🌍 **18 языков**, переключаются на лету; 🌗 **тёмная тема** следует системной
- 📝 **История хранится только в вашем браузере** (IndexedDB) — не видна другим устройствам
- 🪶 **Ни фреймворков, ни сборки** — серверу нужен только `ws`; фронтенд — чистый vanilla JS

## Самостоятельный хостинг

Требуется Node ≥ 18.

```bash
git clone https://github.com/sitboy/lanzap.git zap && cd zap
npm install
node server.js          # по умолчанию слушает :8879, переопределяется через PORT
```

Запускайте за обратным прокси. Три обязательных требования:

- **HTTPS обязателен** (браузеры требуют его для WebRTC / камеры)
- Прокси должен поддерживать **WebSocket upgrade**
- Прокси должен передавать **`X-Real-IP`** (используется для группировки устройств в одной сети)

Пример конфигурации Nginx:

```nginx
server {
    server_name your-domain.example;
    location / {
        proxy_pass http://127.0.0.1:8879;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 3600s;
    }
    # listen 443 ssl; + сертификат (certbot выдаёт его одной командой)
}
```

## Область применения

Только прямое соединение в локальной сети (без STUN/TURN любого рода), поэтому передача
работает **только в пределах одной сети**. Если открыть ссылку из разных сетей, все всё равно
попадут в одну комнату и увидят друг друга, но не смогут установить прямое соединение — интерфейс
чётко сообщает об этом примерно через 10 секунд, вместо бесконечной загрузки.

**Принимать** папки умеет любой браузер, а для **отправки** нужен системный выбор папки: в Safari на iOS его нет, поэтому там этот пункт скрыт. Ограничение — 2000 файлов на папку; при превышении вам прямо предложат сжать её самостоятельно, вместо того чтобы молча потерять часть.

## Как это работает

Сигнализация в `server.js` (`ws`; группирует устройства по исходящему IP / префиксу IPv6‑64,
ручной код комнаты имеет приоритет) плюс фронтенд в `public/` (WebRTC‑меш, чанкинг по 64 КБ с
back‑pressure; сканирование QR предпочитает `BarcodeDetector`, с откатом на jsQR). Дизайн‑система
находится в `design/`.

Папка **не** упаковывается перед отправкой: она идёт как пакет обычных файлов, каждый со своим путём относительно корня, поэтому прогресс по файлам, контроль потока и переподключение работают как прежде. По платформам расходится только последний шаг — запись на диск: File System Access пишет дерево напрямую (потоково, ничего не держится в памяти), все остальные складываются в несжатый ZIP силами `public/zip.js` (~130 строк, без зависимостей). Входящие пути считаются враждебными данными и очищаются, так что выйти за пределы целевой папки невозможно; если диск медленнее сети, принимающая сторона притормаживает отправителя, а не копит очередь в оперативной памяти.
