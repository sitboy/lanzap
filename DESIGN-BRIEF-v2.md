# 近传 Zap · 交互升级设计委托 v2（投给 Claude Design 的 prompt）

> 这是对已有「近传 Zap」设计系统的**扩展/升级**，不是从零重做。请沿用已定稿的品牌、色板、字号、圆角、组件风格（绿色 #10B981 主色、聊天流布局、SVG 图标、双端），只针对下面这批**新交互**出高保真设计与状态穷举。

## 产品一句话（不变）

局域网文件传输助手：同一网络的设备打开同一网址自动成群，像文件传输助手一样互发文字和文件；文字/文件走 P2P 直传，服务器只当接线员、永不碰内容。定位「零安装版 LocalSend」。

## 这次要设计的 6 个新点（核心）

### 1. 默认房 + 私密组队（房间模型）
- **裸链接打开 = 自动进默认大房间**（所有同网设备零操作互见）。首屏不再强调"扫码配对"，而是"你已经在房间里、这些是能连上的设备"。
- **私密组队**：点"＋/组队"→ 生成一个**独特房码 + 二维码 + 可分享链接**；把链接发给某人，对方打开即进这个私密房。请设计"当前在默认房"与"当前在私密房"两种**顶部状态标识**（一眼能分辨、且能一键回默认房）。
- 房码可**自定义成好记的名字**（输入框，已有）。

### 2. 设备列表"只显示连得上的"
- 设备栏/列表**只展示 P2P 真正连得上的设备**（同一局域网）。连接中→短暂"连接中"态；确认连不上（跨网）→**从列表隐去**，不显示"无法直连"的僵尸条目。
- 请设计三种设备状态视觉：**连接中（脉冲/黄）→ 已连接（绿）→（连不上则消失）**。

### 3. 设备身份：独特头像 + 名字（每标签独立）
- 每台设备一个**独特的彩色头像**（由设备 id 派生的稳定色相/identicon，每台一色，所有人看到一致），不再是清一色的蓝。
- 名字不再是光秃秃的 "iPhone"/"Mac"，而是带区分度（如 "iPhone·K2"）。**可点头像自定义名字（和头像颜色/图案）**，改完实时广播给他人。
- 目标体验：**不论同一台机器开了几个标签，每个都是不同头像、不同名字，一眼能区分**。请设计头像体系（自己 vs 他人、手机 vs 电脑、N 种色相示例）+ 自定义身份的编辑面板。

### 4. 群聊（所有人）——要真的可靠、人人同步
- "所有人"是群聊：任何人发的文字，房里每台设备都能看到（后台用 gossip 扩散实现，设计层不用管机制，只需**呈现成一个正常、可靠的群聊**）。
- 请设计群聊消息流：文字气泡、系统提示（谁进来/离开）、时间分隔、每条消息带发送者头像+名字。

### 5. 文件 = "通告—按需拉取"（**这次最重要的新组件**）
- 发送方选文件后，**不立即传**，而是在群聊里**贴出一张"文件通告卡"**：显示 谁分享的、文件名、大小、类型图标。
- 房里每个人都看到这张卡。**谁想要谁点"下载"** → 才开始点对点从发送方拉取 → 卡片进入下载进度 → 完成后变"保存/打开"。
- 请把这张**文件卡的完整状态机**设计出来：
  1. `待下载`（别人分享的，显示"下载"按钮 + 大小）
  2. `下载中`（进度条 + 速率）
  3. `已保存`（可再次打开/保存）
  4. `发送方已离线 / 不可用`（发送方关了页面就拉不到——要明确、友好地告知，而不是转圈）
  5. 自己发出的那张卡（"你分享了 X，已被 N 人下载"之类的回执）
- 图片类文件：卡片可直接出缩略图预览。

### 6. 历史 & 退出
- 聊天记录只存在**本机浏览器**（已有），设计一句轻提示。
- 增加"**退出组队 / 清空本机记录**"入口（可选阅后即焚），请设计确认交互。
