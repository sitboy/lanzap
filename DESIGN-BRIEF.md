# 局域网传输助手 —— UI/UX 设计委托(投给 claude design 的 prompt)

## 产品是什么

一个网页版的"局域网文件传输助手":同一 WiFi 下的设备打开同一个网址,自动出现在同一个会话里,
像用微信文件传输助手一样互发文字和文件。文件走设备之间的 P2P 直连——**不经过服务器、
不限大小、不压缩画质**。免装 App、免登录、免加好友。产品将开源并独立运营推广。

一句话定位:**"不装 App 的 AirDrop + 不用微信的传输助手"**。

## 用户与场景

- 家庭多设备:手机↔电脑互传照片、安装包、文档
- 小微商户:店里多台工作手机之间倒文件
- 办公临时互传:不加好友给同事传个大文件
- 中国用户为主,兼顾海外(中英双语,语言包已外置)

## 交互心智基准(重要)

以**微信文件传输助手**的使用心智为基准:聊天流、左右气泡、文件卡片、底部输入栏——
让中国用户零学习成本。**但这是独立产品**:请建立自有视觉身份(自己的品牌色系、原创图标、
品牌名占位),做到"熟悉但不山寨",不要像素级复刻微信。

## 信息架构(现状,允许你优化重组)

- **移动端(主战场)**:单屏聊天视图 = 顶栏(标题/改名/语言切换) + 本网设备栏(头像横排 +
  "＋邀请"入口) + 聊天流 + 输入栏(文字 + 选文件)
- **桌面端**:左侧窄边栏(设备头像/聊天/邀请/菜单,为未来功能扩展预留竖向图标位) +
  聊天区(顶栏 + 设备栏 + 聊天流 + 工具栏/大输入区/右下发送按钮)
- **组队弹层(双面卡片)**:邀请面 = 二维码 + 5 位大字房码 + 复制链接;
  加入面 = "扫码加入"大按钮 + 房码手输框。手机默认加入面、桌面默认邀请面(角色不对称)
- **全屏扫码视图**:站内调摄像头,取景框 + 提示语 + 取消

## 关键流程(每个状态都需要设计)

1. **主流**:同 WiFi 打开即互见 → 选文件 → 发送中(进度条) → 已送达 / 传输失败(可重试)
2. **组队兜底**:A 点邀请出码 → B 扫码或手输房码 → 进同一房间
3. **状态穷举**:空态(网内只有自己,需引导语和引导视觉)、对方加入/离开提示、
   连接中(黄点)/已连接(绿点)、接收中进度、图片消息(缩略图)、历史记录
   (注明"只存本机浏览器")、摄像头无权限报错
4. **首次进入的价值传达**:新用户 3 秒内明白"这是什么、怎么用"
   (现状是底部一行小字"点对点直连·文件不经过服务器·房间 XXXX",可重新设计)

## 设计交付物

- 移动端(390 宽)与桌面端(880×760 窗口)**全套高保真界面**,覆盖上述全部状态
- **设计 token**:色板(主色/功能色/灰阶)、字号阶梯、间距体系、圆角体系
- **组件规格**:气泡(文字/文件/图片三种)、文件卡片(含文件类型图标体系:PDF/Word/表格/
  压缩包/APK/音视频/未知)、设备头像体系(手机/电脑 × 自己/他人)、弹层、扫码视图、按钮态
- 空态/引导的轻量插画风格(需可用 SVG 手写实现)
- **加分项**:暗色模式;品牌 logo 方向草案;产品命名建议(中文名 + 英文名,
  体现"局域网/直达/快"的气质,中文名要口语可传播)

## 实现约束(务必遵守)

- 纯 Web 实现(vanilla HTML/CSS/JS,无框架无构建),设计必须能用手写 CSS 落地
- 移动优先响应式;同一套信息架构在两端是**两种输入范式**(移动=单行+加号,桌面=工具栏+大输入区)
- 所有文案预留中英双语空间(英文普遍更长)
- 图标一律 SVG(线条或填充,单色可调),不用图片素材
- 性能红线:首屏总重 <60KB,不引入网络字体,不用重度阴影和复杂动效
- 现有实现可参考 https://file.joestudy.net (手机和电脑各开一次看两种形态)
